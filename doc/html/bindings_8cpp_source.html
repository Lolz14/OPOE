2 /*
3  bindings.cpp - Pybind11 bindings for stochastic volatility SDE models, solvers, and option pricers.
4 
5  This module exposes C++ classes and functions to Python, enabling the use of advanced stochastic differential equation (SDE) models,
6  numerical solvers, and option pricing engines in Python workflows. The bindings are designed for flexibility and extensibility,
7  supporting multiple models, solvers, and pricer types.
8 
9  Main Features:
10  --------------
11  - SDE Models:
12  * Heston, Hull-White, Stein-Stein, Jacobi, and Geometric Brownian Motion models.
13  * All models inherit from a common ISDEModel interface.
14  * Model parameters and state dimensions are accessible from Python.
15 
16  - SDE Solvers:
17  * Euler-Maruyama, Milstein, and Interpolated Kahl-Jackel solvers.
18  * Generic binding for solver classes, supporting custom SDE models.
19  * Solvers can simulate SDE paths with optional user-supplied Brownian increments.
20 
21  - Option Payoffs:
22  * Base IPayoff interface for extensibility.
23  * European call and put payoffs with strike getter/setter and evaluation methods.
24 
25  - Option Pricers:
26  * Monte Carlo (MCOptionPricer), FFT-based (FFTOptionPricer), and Operator Expansion (OPEOptionPricer) pricers.
27  * OPEOptionPricer is templated for different expansion orders (N = 3, 5, 7, 9, 10).
28  * All pricers accept model, payoff, and numerical method parameters.
29 
30  - Enumerations:
31  * QuadratureMethod: TanhSinh, QAGI.
32  * SolverType: EulerMaruyama, Milstein, IJK.
33 
34  Usage:
35  ------
36  Import the module in Python as `opoe` and instantiate models, solvers, payoffs, and pricers as needed.
37  Example:
38  import opoe
39  model = opoe.HestonModel(...)
40  payoff = opoe.EuropeanCallPayoff(K=100)
41  pricer = opoe.MCOptionPricer(ttm=1.0, rate=0.05, payoff=payoff, model=model)
42  price = pricer.price()
43 
44  Notes:
45  ------
46  - All numerical types (Real, Matrix, Vector) are defined via the traits system for flexibility.
47  - Shared pointers are used for memory management and Python interoperability.
48  - The bindings are designed to be extensible for additional models, solvers, and pricer types.
49 
50  Author: [Your Name or Organization]
51  License: [Appropriate License]
52 */
